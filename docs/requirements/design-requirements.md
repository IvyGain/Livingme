## 1. 美的方向性

### 1.1 コアコンセプト

> **「愛のあるこころの居場所」** —— リビングのような温かさと安心感

- **トーン**: warm-intimate（温かく・親密に）
- **ムード**: nurturing（育み・包み込む）
- **方向性**: organic-warmth（有機的な温もり）

### 1.2 ビジュアルキーワード

| キーワード | 表現 |
|-----------|------|
| 温かさ | アンバー・テラコッタ系の暖色 |
| 安心感 | 余白・ゆったりしたレイアウト |
| 自然 | 有機的な形・ナチュラルテクスチャ |
| 愛 | 柔らかい曲線・ぬくもりのある書体 |
| 落ち着き | くすみカラー・ローコントラスト |

### 1.3 差別化ポイント

情報を詰め込まず、**呼吸できる空間**を持ったデザイン。
ユーザーがサイトを開くたびに「ただいま」と感じられる居場所。

## 3. UIコンポーネント方針

### 3.1 コンポーネントライブラリ

| 項目 | 選定 |
|------|------|
| ベースライブラリ | **shadcn/ui** |
| スタイリング | **Tailwind CSS v4** |
| アニメーション | **motion/react** |
| アイコン | **lucide-react** |
| ユーティリティ | **cn() (clsx + tailwind-merge)** |

### 3.2 コンポーネント設計原則

- **温かみ優先**: シャープな角よりも丸みのある角丸（border-radius: 12px〜16px）
- **余白を大切に**: コンテンツが詰まった印象を避ける
- **カード中心**: コンテンツはカード形式でグルーピング
- **状態の明確化**: ホバー・フォーカス・アクティブ状態を丁寧に設計

### 3.3 インタラクション原則

| 原則 | 実装 |
|------|------|
| アニメーション制限 | 200ms以下のインタラクション |
| パフォーマンス | Compositor props のみ（transform, opacity）|
| 入力フィードバック | 即時視覚フィードバック（100ms以内）|
| 禁止エフェクト | グロー効果・過剰ドロップシャドウ・レイアウトプロパティのアニメーション |

### 3.4 主要コンポーネント仕様

#### ボタン

| バリアント | 用途 | スタイル |
|-----------|------|---------|
| Primary | メインCTA | Warm Chestnut 背景、白テキスト |
| Secondary | サブアクション | Cream Linen 背景、Warm Chestnut テキスト |
| Ghost | ナビゲーション | 透明背景、ホバーで薄背景 |
| Destructive | 削除・警告 | エラーカラー |

#### カード

- 背景: `#FFFFFF` または `#F5EDD8`
- ボーダー: 1px `#B8A99A`（薄い）または ボーダーなし
- 角丸: 12px〜16px
- シャドウ: `0 2px 8px rgba(61, 43, 31, 0.08)` — 軽い影

## 4. レスポンシブ方針

### 4.1 モバイルファースト

**モバイルファースト**で設計。会員の主要アクセスはスマートフォンを想定。

### 4.2 ブレークポイント

| 名前 | サイズ | 対象 |
|------|--------|------|
| `sm` | 640px | 大きなスマートフォン |
| `md` | 768px | タブレット縦 |
| `lg` | 1024px | タブレット横 / 小型PC |
| `xl` | 1280px | デスクトップ |
| `2xl` | 1536px | 大型デスクトップ |

### 4.3 モバイル優先レイアウト

| 要素 | モバイル | タブレット | デスクトップ |
|------|---------|-----------|-------------|
| ナビゲーション | ボトムバー | サイドバー | サイドバー |
| コンテンツカード | 1カラム | 2カラム | 3カラム |
| 管理画面 | スタック | テーブル | テーブル |
| カレンダー | 週表示デフォルト | 月表示 | 月表示 |

### 4.4 タッチ最適化

- タッチターゲット: 最小 44px × 44px
- スワイプジェスチャー: カード・カレンダー
- ボトムシート: モバイルのモーダル代替

## 5. レイアウト原則

### 5.1 スペーシングスケール

| トークン | サイズ | 用途 |
|---------|--------|------|
| `xs` | 4px | 密接した要素間 |
| `sm` | 8px | コンポーネント内 |
| `md` | 16px | 要素間 |
| `lg` | 24px | セクション内 |
| `xl` | 32px | セクション間 |
| `2xl` | 48px | 大セクション間 |
| `3xl` | 64px | ページレベル |

### 5.2 レイアウトルール

- **最大幅**: コンテンツエリア `max-w-5xl` (1024px)
- **サイドバー**: `280px` 固定（デスクトップ）
- **高さ単位**: `h-dvh`（dynamic viewport height）を使用
- **グリッド**: CSS Gridベース、Flexboxはコンポーネント内

### 5.3 禁止レイアウトパターン

- **禁止**: 予測可能な3カラム等幅グリッド（AI slop）
- **禁止**: 過剰なドロップシャドウの積み重ね
- **禁止**: z-indexの乱用（管理しきれないスタッキング）

## 7. 画像・メディア方針

- **サムネイル比率**: 16:9（動画）、3:2（コラム）
- **最適化**: Next.js `<Image>` コンポーネント使用必須
- **フォーマット**: WebP優先、フォールバックJPEG
- **遅延読み込み**: `loading="lazy"` / Intersection Observer

## 10. Anti-Patterns（絶対禁止）

| カテゴリ | 禁止パターン | 理由 |
|---------|------------|------|
| フォント | Inter, Roboto, Arial | AI slopの典型 |
| カラー | 紫グラデーション on 白背景 | AI slopの典型 |
| カラー | ビビッドなブルー系ビジネスカラー | ブランドと不一致 |
| レイアウト | 予測可能な等幅グリッド | 差別化不足 |
| エフェクト | グロー効果 | AI slopの典型 |
| エフェクト | 過剰なドロップシャドウ | AI slopの典型 |
| アニメーション | レイアウトプロパティのアニメーション | パフォーマンス問題 |
| 情報密度 | コンテンツの詰め込み | ブランドコンセプトと矛盾 |
