## はじめに読んでください

### この手順書でできること

この手順書を読むと、**Living Me の会員サイトで使うコンテンツデータベース**を
Lark（ラーク）というアプリを使って設計・構築できるようになります。

構築するのは、以下の4つのデータベースです。

| データベース名 | 何を管理するか |
|--------------|-------------|
| アーカイブ管理 | 朝会・夜会などの動画記録 |
| イベント管理 | 今後の開催予定 |
| 今日のコンテンツ | 毎日のエネルギーシェア・テーマ |
| コラム管理 | 主宰者のコラム・つぶやき |

### Lark Base（多次元テーブル）とは？

```
Lark Base = Excelの表 + データベースの機能 を組み合わせたもの

普通のExcelとの違い:
┌────────────────┬────────────────────────────────────┐
│  Excel（普通）  │ 入力ミスを防げない、複数人で使いにくい │
│  Lark Base     │ 入力タイプを制限できる、複数人で同時編集可 │
└────────────────┴────────────────────────────────────┘

エンジニアが使うデータベースとの違い:
┌────────────────────┬──────────────────────────────────┐
│  エンジニア向けDB   │ コードを書かないと操作できない        │
│  Lark Base         │ 画面を見ながらクリックするだけで操作可 │
└────────────────────┴──────────────────────────────────┘
```

### 役割分担（重要）

Living Me システムでは、データを2か所に分けて管理しています。

```
┌─────────────────────────────┐    ┌──────────────────────────────┐
│   PostgreSQL（エンジニア管理）│    │   Lark Base（運営スタッフ管理）│
│                             │    │                              │
│  ・会員情報（氏名・メール）   │    │  ・アーカイブ動画の情報       │
│  ・ログイン・パスワード情報   │    │  ・イベントスケジュール       │
│  ・決済・課金情報            │    │  ・毎日のエネルギーシェア     │
│  ・個人のジャーナル日記      │    │  ・コラム・つぶやき           │
│                             │    │                              │
│  ⚠️ 個人情報なので           │    │  ✅ 非エンジニアが              │
│  エンジニアのみ触る          │    │  Larkから自由に編集可能        │
└─────────────────────────────┘    └──────────────────────────────┘
```

> **ポイント**: この手順書では「右側」のLark Baseを構築します。

## 1. 事前準備

### 1-1. 必要なもの

```
└────────────────────────────────────┘
```

## 2. Lark Base（アプリ）の作成

> **アナロジー**: 「Base（ベース）」は「Excelファイル」にあたります。
> その中に複数の「シート（テーブル）」を作っていくイメージです。

### 2-1. 新しいBaseを作成する

**手順:**

1. 左サイドバーの「**ドキュメント**」をクリック
2. 画面左上の「**+新規**」ボタンをクリック
3. メニューから「**多次元テーブル（Base）**」を選択

   ```
   ┌──────────────────────────────┐
   │  新規作成                     │
   │  ─────────────────────────── │
   │  📄 ドキュメント              │
   │  📊 多次元テーブル（Base） ← │
   │  📋 スプレッドシート          │
   └──────────────────────────────┘
   ```

4. 名前の入力欄に「**LivingMe コンテンツ管理**」と入力
5. 「**作成**」ボタンをクリック

### 2-2. 初期画面の確認

作成すると、「テーブル1」という名前の空のテーブルが表示されます。

```
LivingMe コンテンツ管理
├── テーブル1  ← 最初からある（名前を変更する）
└── ＋ テーブルを追加
```

### 2-3. 最初のテーブルの名前を変更する

1. 上部の「**テーブル1**」タブを**ダブルクリック**
2. テキストが選択状態になる
3. 「**アーカイブ**」と入力してEnterキーを押す

## 3. テーブル1: アーカイブ管理

> **このテーブルで管理するもの**: 朝会・夜会・学習コンテンツなどの動画記録

### テーブルの全体像

```
アーカイブテーブルのイメージ:

| タイトル          | 日付       | カテゴリ | 動画URL   | 公開 |
|-----------------|-----------|---------|---------|------|
| 朝会 - 意図設定   | 2026-03-20 | 朝会    | zoom... | ✅   |
| 夜会 - 振り返り   | 2026-03-19 | 夜会    | zoom... | ✅   |
| エネルギー入門    | 2026-03-15 | 学習    | zoom... | ✅   |
```

### 3-1. 既存の列を確認・削除する

最初から「フィールド1」などの列が作られている場合は、必要なものだけ残します。

1. 列ヘッダーを**右クリック**
2. 「**フィールドを削除**」をクリック
3. 確認ダイアログで「**削除**」をクリック

> **ヒント**: 最初にある「タイトル」フィールドは必ず残しておいてください。

### 3-2. 「タイトル」フィールドの設定

最初から「タイトル」列があります。これを確認します。

1. 「タイトル」列のヘッダーをクリック
2. フィールドの設定画面が開く
3. タイプが「**テキスト**」になっていることを確認
4. 変更不要であれば「**保存**」をクリック

### 3-3. 「日付」フィールドを追加する

1. 一番右の「**＋**」ボタンをクリック（列を追加）
2. フィールド名に「**日付**」と入力
3. フィールドタイプの一覧から「**日付**」を選択

   ```
   フィールドタイプの選択肢:
   ┌────────────────────────────────────────┐
   │ 📝 テキスト                            │
   │ 🔢 数値                               │
   │ 🔘 単一選択      ← カテゴリで使う     │
   │ ☑️ チェックボックス ← 公開フラグで使う │
   │ 📅 日付          ← ここを選択         │
   │ 🕐 日時                               │
   │ 🔗 URL           ← 動画URLで使う      │
   │ 📎 添付ファイル   ← 画像で使う        │
   └────────────────────────────────────────┘
   ```

4. 日付フォーマットは「**YYYY/MM/DD**」を選択
5. 「**保存**」をクリック

### 3-4. 「カテゴリ」フィールドを追加する

1. 「**＋**」ボタンをクリック
2. フィールド名に「**カテゴリ**」と入力
3. フィールドタイプで「**単一選択**」を選択
4. 以下の選択肢を追加する（「**＋ 選択肢を追加**」をクリック）:

   | 選択肢名 | 色（任意） |
   |---------|---------|
   | 朝会 | 🟡 黄色 |
   | 夜会 | 🔵 青 |
   | 学習 | 🟢 緑 |
   | イベント | 🟠 オレンジ |
   | その他 | ⚪ グレー |

5. 「**保存**」をクリック

> **注意**: 単一選択は「1つしか選べない」タイプです。
> 複数選択したい場合は「**複数選択**」を使います（タグで使用します）。

### 3-5. 「説明」フィールドを追加する

1. 「**＋**」ボタンをクリック
2. フィールド名に「**説明**」と入力
3. フィールドタイプで「**テキスト**」を選択
4. 「**保存**」をクリック

### 3-6. 「動画URL」フィールドを追加する

1. 「**＋**」ボタンをクリック
2. フィールド名に「**動画URL**」と入力
3. フィールドタイプで「**URL**」を選択
4. 「**保存**」をクリック

> **使い方**: ZoomのクラウドレコーディングURL、Lark DriveのファイルURL、YouTubeのURLなど、何でも貼り付けられます。

### 3-7. 「サムネイル」フィールドを追加する

1. 「**＋**」ボタンをクリック
2. フィールド名に「**サムネイル**」と入力
3. フィールドタイプで「**添付ファイル**」を選択
4. 「**保存**」をクリック

> **使い方**: アーカイブのカバー画像（サムネイル）をここに添付します。

### 3-8. 「議事録（分数）」フィールドを追加する

1. 「**＋**」ボタンをクリック
2. フィールド名に「**議事録（分数）**」と入力
3. フィールドタイプで「**数値**」を選択
4. 「**保存**」をクリック

> **使い方**: 動画の長さを分で入力します（例: 45）。

### 3-9. 「要約」フィールドを追加する

1. 「**＋**」ボタンをクリック
2. フィールド名に「**要約**」と入力
3. フィールドタイプで「**テキスト**」を選択
4. 「**保存**」をクリック

> **将来の機能**: AIが自動で生成する予定のフィールドです。

### 3-10. 「エネルギーシェア」フィールドを追加する

1. 「**＋**」ボタンをクリック
2. フィールド名に「**エネルギーシェア**」と入力
3. フィールドタイプで「**テキスト**」を選択
4. 「**保存**」をクリック

> **使い方**: 当日のエネルギーシェアの文章を入力します。

### 3-11. 「ジャーナリングテーマ」フィールドを追加する

1. 「**＋**」ボタンをクリック
2. フィールド名に「**ジャーナリングテーマ**」と入力
3. フィールドタイプで「**テキスト**」を選択
4. 「**保存**」をクリック

> **使い方**: 「今日あなたが大切にしたいことは？」などのテーマを入力します。

### 3-12. 「タグ」フィールドを追加する

1. 「**＋**」ボタンをクリック
2. フィールド名に「**タグ**」と入力
3. フィールドタイプで「**複数選択**」を選択
4. 以下の選択肢を追加:

   | タグ名 |
   |-------|
   | マインドフルネス |
   | エネルギー |
   | 朝ルーティン |
   | 内観 |
   | セルフケア |
   | コミュニティ |
   | 学び |

5. 「**保存**」をクリック

> **使い方**: 複数のタグを選択できます（例: 朝ルーティン + マインドフルネス）。

### 3-13. 「公開」フィールドを追加する（重要）

1. 「**＋**」ボタンをクリック
2. フィールド名に「**公開**」と入力
3. フィールドタイプで「**チェックボックス**」を選択
4. 「**保存**」をクリック

> **重要**: このチェックをONにすると会員サイトに表示されます。
> 準備ができるまでは必ずOFFのままにしてください。

### 3-14. 「公開日時」フィールドを追加する

1. 「**＋**」ボタンをクリック
2. フィールド名に「**公開日時**」と入力
3. フィールドタイプで「**日時**」を選択
4. フォーマットを「**YYYY/MM/DD HH:mm**」に設定
5. 「**保存**」をクリック

> **使い方**: 未来の日時を設定すると、その時刻から自動で公開されます。

### 3-15. アーカイブテーブルの完成確認

以下のフィールドが全て揃っていることを確認してください。

```
✅ 完成したアーカイブテーブルのフィールド一覧:

[ ] タイトル        （テキスト）     必須
[ ] 日付            （日付）         必須
[ ] カテゴリ        （単一選択）     必須
[ ] 説明            （テキスト）
[ ] 動画URL         （URL）
[ ] サムネイル      （添付ファイル）
[ ] 議事録（分数）  （数値）
[ ] 要約            （テキスト）
[ ] エネルギーシェア（テキスト）
[ ] ジャーナリングテーマ（テキスト）
[ ] タグ            （複数選択）
[ ] 公開            （チェックボックス）必須
[ ] 公開日時        （日時）
```

## 4. テーブル2: イベント管理

#### フィールド④: 終了日時
- フィールド名: 「**終了日時**」
- タイプ: 「**日時**」
- フォーマット: 「YYYY/MM/DD HH:mm」

#### フィールド⑤: 説明
- フィールド名: 「**説明**」
- タイプ: 「**テキスト**」

#### フィールド⑥: 参加URL
- フィールド名: 「**参加URL**」
- タイプ: 「**URL**」

> **使い方**: ZoomやGoogle MeetのURLを入力します。

#### フィールド⑦: 開催場所

- タイプ: 「**テキスト**」

> **使い方**: オフラインイベントの住所を入力します（オンラインは空欄でOK）。

#### フィールド⑧: 定員

## 6. テーブル4: コラム管理

### 6-2. フィールドを追加する

- タイプ: 「**テキスト**」

> **ヒント**: 長い文章を書く場合は、セルをクリックして展開して入力します。

#### フィールド③: 著者名
- フィールド名: 「**著者名**」
- タイプ: 「**テキスト**」

> **使い方**: 「田中美咲」など著者の名前を入力します。

#### フィールド④: 公開日時
- フィールド名: 「**公開日時**」
- タイプ: 「**日時**」

#### フィールド⑤: 公開
- フィールド名: 「**公開**」
- タイプ: 「**チェックボックス**」

### 6-3. コラムテーブルの完成確認

```
✅ 完成したコラムテーブルのフィールド一覧:

[ ] タイトル   （テキスト）         必須
[ ] 本文       （テキスト）         必須
[ ] 著者名     （テキスト）         必須
[ ] 公開日時   （日時）             必須
[ ] 公開       （チェックボックス） 必須
```

## 7. テーブル間のリレーション設定

> **リレーションとは?**: 複数のテーブルを「つなげる」機能です。
> 例えば「今日のコンテンツ」と「アーカイブ」をつなぐことで、
> 「今日の内容は〇月〇日の朝会アーカイブから」と関連付けできます。

### 7-1. 「今日のコンテンツ」と「アーカイブ」をつなぐ

1. 「**今日のコンテンツ**」テーブルを開く
2. 「**＋**」で新しいフィールドを追加
3. フィールド名に「**元のアーカイブ**」と入力
4. フィールドタイプで「**リンクされたレコード**」を選択

   ```
   フィールドタイプの選択（スクロールして探す）:

   ┌──────────────────────────────────────────┐
   │  ...                                     │
   │  🔗 リンクされたレコード  ← これを選択    │
   │  ...                                     │
   └──────────────────────────────────────────┘
   ```

5. 「**リンク先テーブル**」の選択で「**アーカイブ**」を選択
6. 「**保存**」をクリック

これで「今日のコンテンツ」の各レコードから、
対応するアーカイブ動画を直接参照できるようになります。

## 8. ビュー（見やすい表示）の設定

> **ビューとは?**: 同じデータを「別の見せ方」で表示する機能です。
> データ自体は変わらず、見た目だけが変わります。

### 8-1. アーカイブテーブルに「カレンダービュー」を追加する

1. 「**アーカイブ**」テーブルを開く
2. 左サイドの「**ビューを追加**」または「**＋**」をクリック
3. 「**カレンダー**」を選択
4. ビュー名に「**月別カレンダー**」と入力
5. 「日付フィールド」で「**日付**」を選択
6. 「**作成**」をクリック

これで、アーカイブがカレンダー形式で表示されます。

### 8-2. イベントテーブルに「ガントチャートビュー」を追加する

1. 「**イベント**」テーブルを開く
2. 「**ビューを追加**」をクリック
3. 「**ガントチャート**」を選択
4. ビュー名に「**スケジュール**」と入力
5. 開始日フィールドで「**開始日時**」を選択
6. 終了日フィールドで「**終了日時**」を選択
7. 「**作成**」をクリック

### 8-3. 「公開済み」フィルタービューを作成する

各テーブルに「公開チェック済みのレコードだけ表示する」ビューを作ります。

1. 「**アーカイブ**」テーブルを開く
2. 「**ビューを追加**」→「**グリッド（表）**」を選択
3. ビュー名に「**公開済み一覧**」と入力
4. 「**作成**」をクリック
5. 上部の「**フィルター**」ボタンをクリック
6. 「**条件を追加**」をクリック
7. フィールドで「**公開**」を選択
8. 条件で「**チェック済み**」を選択
9. 「**保存**」をクリック

これで「公開済みのアーカイブのみ」が表示されます。

## 9. 動作確認とテストデータ入力

### 9-1. テストデータを入力する

設計が完了したら、各テーブルに1件ずつテストデータを入力しましょう。

#### アーカイブのテストデータ

1. 「**アーカイブ**」テーブルを開く
2. 空の行をクリックして以下を入力:

   ```
   タイトル        : テスト朝会 - 意図設定
   日付            : 今日の日付を選択
   カテゴリ        : 朝会
   説明            : これはテスト用のデータです
   エネルギーシェア: 今日は穏やかなエネルギーを感じています
   ジャーナリングテーマ: 今日、私が大切にしたいことは何ですか？
   公開            : チェックしない（テストなのでOFF）
   ```

#### イベントのテストデータ

1. 「**イベント**」テーブルを開く
2. 空の行をクリックして以下を入力:

   ```
   タイトル       : テスト朝会 - 水曜日
   イベント種別   : 朝会
   開始日時       : 明日の07:00
   終了日時       : 明日の08:00
   参加URL        : https://zoom.us/j/example（仮のURL）
   公開           : チェックしない（テストなのでOFF）
   ```

#### 今日のコンテンツのテストデータ

1. 「**今日のコンテンツ**」テーブルを開く
2. 空の行をクリックして以下を入力:

   ```
   日付                : 今日の日付（例: 2026-03-22）
   エネルギーシェア    : 今日は穏やかな春の風のようなエネルギーです
   ジャーナリングテーマ: 私が今、本当に大切にしたいことは何ですか？
   朝会メモ            : 今日の朝会では自分軸について話しました
   公開                : チェックしない（テストなのでOFF）
   ```

### 9-2. 入力内容の確認ポイント

テストデータを入力したら、以下を確認してください。

```
✅ 確認チェックリスト:

[ ] アーカイブテーブルに1件以上のデータが入っている
[ ] イベントテーブルに1件以上のデータが入っている
[ ] 今日のコンテンツテーブルに今日の日付で1件入っている
[ ] 単一選択フィールドで選択肢が正しく表示される
[ ] チェックボックスがON/OFFできる
[ ] 日付フィールドでカレンダーが開いて選択できる
[ ] URLフィールドに入力したURLがクリックで開ける
```

## 10. エンジニアへの引き渡し情報の確認

> **この章は重要です**: エンジニアがシステムとLark Baseをつなぐために
> 必要な「ID」を確認して共有する必要があります。

### 10-1. Base App Token（ベース全体のID）の確認

1. 「LivingMe コンテンツ管理」Baseを開く
2. ブラウザのURL欄を確認する

   ```
   URL例:
   https://www.larksuite.com/base/AbCdEfGhIjKlMnOp/...
                                    ↑
                             これが App Token
                             （「AbCdEfGhIjKlMnOp」の部分）
   ```

3. この値をメモしておく（例: `AbCdEfGhIjKlMnOp`）

### 10-2. 各テーブルのTable IDの確認

各テーブルのIDをエンジニアに伝える必要があります。

**方法1: URLから確認する**
1. 各テーブルを開く
2. URLを確認する
   ```
   URL例:
   https://www.larksuite.com/base/AbCdEfGhIjKlMnOp?table=tblXxXxXxXxXxXx&...
                                                          ↑
                                                    これが Table ID
   ```

**方法2: API設定から確認する（推奨）**
1. Baseの右上「・・・」メニューをクリック
2. 「API」または「開発者ツール」を選択
3. テーブル一覧でIDを確認

### 10-3. エンジニアへ共有する情報まとめ

以下の情報をスプレッドシートやメモにまとめてエンジニアに共有してください。

```
【Lark Base 設定情報】

Base App Token: ________________________
（URLの /base/ の後ろの部分）

テーブルID一覧:
- アーカイブ      テーブルID: ________________________
- イベント        テーブルID: ________________________
- 今日のコンテンツ テーブルID: ________________________
- コラム          テーブルID: ________________________
```

### 10-4. Lark Open Platform の設定（エンジニアが行う作業）

> **注意**: この項目はエンジニアに依頼してください。自分で設定しなくてOKです。

エンジニアには以下の作業をお願いします:

1. Lark Open Platformでアプリを作成
2. 作成したBaseにアプリの権限を付与
3. 環境変数（App ID / App Secret）の設定

## 11. 日常の運用手順

### 11-1. 毎日の運用フロー

```
【朝会がある日の流れ】

07:00  朝会実施（Zoom等）
          ↓
09:30  動画を Lark Drive にアップロード
          ↓
09:35  アーカイブテーブルに新しい行を追加
          ↓
09:40  各フィールドを入力する:
        - タイトル（例: 2026-03-22 朝会アーカイブ）
        - 日付（今日の日付）
        - カテゴリ（朝会）
        - 動画URL（Lark DriveのリンクまたはZoomのURL）
        - エネルギーシェア（今朝のエネルギー文章）
        - ジャーナリングテーマ（今日のテーマ）
          ↓
09:45  今日のコンテンツテーブルに今日の行を追加/更新
          ↓
09:50  内容を確認したら「公開」チェックをON
          ↓
10:00  会員サイトに自動反映（最大5分）✅
```

### 11-2. アーカイブを公開する手順（詳細）

1. 「**アーカイブ**」テーブルを開く
2. 該当のアーカイブ行を探す
3. 全てのフィールドが正しく入力されていることを確認
4. 「**公開**」列のチェックボックスをクリックしてONにする
5. 5分以内に会員サイトへ反映

### 11-3. イベントを登録する手順（詳細）

1. 「**イベント**」テーブルを開く
2. 最後の行の空白セルをクリック（または「**＋行を追加**」をクリック）
3. 以下を入力:
   - **タイトル**: イベント名
   - **イベント種別**: 種類を選択
   - **開始日時**: 日時を設定（クリックすると日時選択カレンダーが開く）
   - **参加URL**: ZoomのURLを貼り付け
4. 「**公開**」にチェック

### 11-4. 今日のエネルギーシェアを更新する手順

1. 「**今日のコンテンツ**」テーブルを開く
2. 今日の日付の行を探す（ない場合は新しく追加）
3. 「**エネルギーシェア**」セルをクリックして入力
4. 「**ジャーナリングテーマ**」も入力
5. 「**公開**」にチェックをON

> **ヒント**: 「公開」チェックを一度ONにすると会員サイトに即座に反映されます。
> 内容を修正したい場合は、そのまま編集すれば自動で更新されます。

### 11-5. 動画をLark Driveにアップロードする手順

1. Larkアプリを開く
2. 左サイドバーの「**ドライブ**」をクリック
3. 「**LivingMe 動画アーカイブ**」フォルダを開く
4. 該当カテゴリのフォルダ（例: 朝会）→ 年月フォルダ（例: 2026-03）に移動
5. 「**アップロード**」ボタンをクリック
6. 動画ファイルを選択してアップロード
7. アップロード完了後、ファイルを右クリック
8. 「**リンクをコピー**」をクリック
9. アーカイブテーブルの「動画URL」欄に貼り付け

## 12. よくあるミスと対処法

### ❌ ミス1: 公開チェックを忘れた

**症状**: Lark Baseに入力したのに会員サイトに表示されない
**対処**: 「公開」チェックボックスがONになっているか確認する

```
確認手順:
1. 該当テーブルを開く
2. 対象レコードの「公開」列を確認
3. □（チェックなし）→ ✅（チェックあり）になっているか確認
```

---

### ❌ ミス2: 日付の形式が間違っている

**症状**: 日付フィールドに手入力したが、カレンダーに表示されない
**対処**: 日付はキーボード入力せず、必ずカレンダーから選択する

```
正しい操作:
1. 日付セルをクリック
2. カレンダーアイコン（📅）をクリック
3. カレンダーから日付を選択
4. 確定ボタンをクリック
```

---

### ❌ ミス3: 同じ日付の「今日のコンテンツ」を2つ作ってしまった

**症状**: 同じ日付のレコードが2件ある
**対処**: 古い方（内容が少ない方）を削除する

```
削除手順:
1. 削除したい行を右クリック
2. 「行を削除」をクリック
3. 確認ダイアログで「削除」をクリック
```

---

### ❌ ミス4: 間違ったデータを公開してしまった

**症状**: 不完全な内容が会員サイトに表示されてしまった
**対処**: 「公開」チェックをOFFにしてすぐに非公開にする

```
緊急対処:
1. 該当テーブルを開く
2. 問題のレコードを探す
3. 「公開」チェックをクリックしてOFFにする
4. 5分以内に会員サイトから消える
```

---

### ❌ ミス5: テーブルを間違えて操作した

**症状**: 別のテーブルのデータを編集してしまった
**対処**: 画面上部のタブで現在どのテーブルを開いているか確認する

```
テーブルの確認:
画面上部のタブを確認する
┌──────────────────────────────────────────────────────┐
│ アーカイブ | イベント | 今日のコンテンツ | コラム       │
│    ↑                                                  │
│  現在このタブが選ばれているテーブルを操作しています    │
└──────────────────────────────────────────────────────┘
```

---

### ❌ ミス6: URLを貼り付けたがクリックできない

**症状**: 動画URLを入力したが、URLフィールドでクリックしても開かない
**対処**: URLフィールドのタイプが「URL」になっているか確認する

```
確認手順:
1. 列ヘッダー「動画URL」をクリック
2. フィールドの設定が開く
3. タイプが「URL」になっているか確認
4. テキストになっている場合は「URL」に変更して保存
```
