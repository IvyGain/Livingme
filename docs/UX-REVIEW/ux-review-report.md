# Living Me UX レビューレポート

**実施日**: 2026-03-20
**対象**: Phase 2 設計書全体
**レビュアー**: UX Review Agent

---

## エグゼクティブサマリー

Living Me の設計は「リビングのような温かさ」というコンセプトを一貫して反映できている。
会員の日課化（毎日トップページを開く習慣）と情緒的つながりを促進する設計として、概ね良好。
以下に改善提案と確認事項を示す。

**総合スコア**: 83 / 100

---

## 1. ユーザーフロー評価

### 1.1 初回ログインフロー ✅ Good

```
Discord ログイン → ステータス判定 → 適切なページへ
```

- Discord OAuth 1本化は**摩擦を最小化**する良い判断
- trial ユーザーが入会案内に自然にたどり着ける設計

### 1.2 日課化フロー（毎日の導線）✅ Good

```
アプリ起動 → トップページ → エネルギーシェア確認
                         → ジャーナリングテーマ確認 → 書く
```

- エネルギーシェアとジャーナリングテーマを大きく配置する設計は正しい
- **改善提案**: 「おはようございます、〇〇さん」等の時間帯別挨拶を追加すると、
  帰ってきた感を強化できる

### 1.3 アーカイブ探索フロー ⚠️ 要検討

```
アーカイブ一覧 → 検索/フィルター → 動画詳細
```

**問題点**: 会員が「何を見たいかわからない」状態でアーカイブを訪れる可能性がある。
キーワード検索に依存しすぎると探索が困難。

**改善提案**:
- 「今日のテーマに関連するアーカイブ」のレコメンドセクションを追加
- 「はじめにおすすめ」キュレーションカテゴリ
- 再生履歴に基づく「続きを見る」セクション（将来）

### 1.4 管理者の今日更新フロー ✅ Good

```
/admin/today → 入力 → AI抽出（P1）→ プレビュー → 公開
```

- 非エンジニア向けに5分以内完結の設計は適切
- **確認事項**: AI 抽出が失敗した場合の手動フォールバックを必ず実装すること

## 2. 情報アーキテクチャ評価

### 2.1 ナビゲーション構造

**モバイル BottomNav 5タブ構成**:

| タブ | 評価 |
|------|------|
| ホーム | ✅ 明確 |
| アーカイブ | ✅ 明確 |
| イベント | ✅ 明確 |
| ジャーナル | ✅ 明確 |
| メニュー | ⚠️ 「メニュー」は汎用的すぎる |

**改善提案**: 5タブ目を「メニュー」ではなく「わたし」（プロフィール・申請フォーム・アンバサダー）にすることで、
会員が自分の活動を管理しやすくなる。

### 2.2 コンテンツの分類 ✅ Good

- アーカイブカテゴリ（朝会・夜会・学習・イベント）は適切
- タグ検索とカテゴリ検索の併用は必要

### 2.3 申請フォームの導線 ⚠️ 要改善

申請フォームの入口が不明確。
現在: `/(member)/forms/[slug]` への直接アクセスが想定されているが、
会員がどこからフォームを発見するかが設計されていない。

**改善提案**:
- トップページまたはメニューに「申請・手続き」セクションを追加
- 各フォームは対象権限に応じて表示/非表示

## 3. アクセシビリティ評価

### 3.1 カラーコントラスト ✅ Pass

| 組み合わせ | コントラスト比 | 判定 |
|-----------|-------------|------|
| `#3D2B1F` on `#FDF9F3` | 8.9:1 | ✅ AAA |
| `#FFFFFF` on `#8B5E3C` | 5.2:1 | ✅ AA |
| `#7A6155` on `#FDF9F3` | 4.7:1 | ✅ AA |
| `#B8A99A` on `#FDF9F3` | 2.6:1 | ❌ 失敗（装飾のみ使用すること）|

**注意**: `#B8A99A` (Warm Taupe) は**テキストに使用してはならない**（プレースホルダー・ボーダーのみ）。

### 3.2 タッチターゲット ✅ Good

- ボタン最小高さ 44px の定義は適切
- カード全体をタップ可能にする設計（`<Link>` ラップ）は正しい

### 3.3 フォーカス管理 ⚠️ 要実装

設計書にフォーカストラップ・フォーカス移動の仕様が不足。

**改善提案**:
- モーダル/ボトムシート開閉時のフォーカス管理を仕様に追加
- スキップナビゲーション（`#main-content` へのリンク）を実装

## 4. モバイルUX評価

### 4.1 PWA体験 ✅ Good

- ホーム画面追加 + ボトムナビの組み合わせはネイティブライクで良い
- Safe Area 対応の定義は適切

### 4.2 スクロール体験 ✅ Good

- アーカイブ横スクロール（snap）は「続きがある」を示す良いパターン
- 縦スクロールメインでシンプルな構造

### 4.3 オフライン体験 ⚠️ 軽微

サービスワーカー定義はあるが、キャッシュ戦略が未定義。

**改善提案**: 最低限、直前に見たトップページコンテンツをキャッシュする

---

## 5. 感情設計評価

### 5.1 ウェルカム体験 ⚠️ 強化推奨

現在の設計では機能的なコンテンツ表示に集中しているが、
**感情的なつながりを作る要素**がやや薄い。

**改善提案**:
```
ログイン直後の初回訪問:
  「〇〇さん、おかえりなさい」
  今日のエネルギーシェア → 大きく・温かく

連続ログイン時:
  「3日連続でジャーナリングしていますね、素晴らしいです」
  （将来機能だが、設計時点で余白を確保すること）
```

### 5.2 エラー・空状態の設計 ⚠️ 未設計

空状態（データなし）・エラー状態のUXが設計書に含まれていない。

**改善提案**: 以下の空状態コピーを定義すること:

| 状態 | コピー案 |
|------|---------|
| 今日のコンテンツ未設定 | 「今日の準備中です。もうしばらくお待ちください ✨」|
| アーカイブ検索0件 | 「見つかりませんでした。別のキーワードを試してみてください」|
| ジャーナリング初回 | 「最初の一歩を踏み出しましょう。今日のテーマを見てみてください」|

## 6. 管理画面UX評価

### 6.1 非エンジニア向け設計 ✅ 意識あり

- 今日の更新フローが5分以内完結を目標としている点は正しい
- 「AI抽出」ボタンの存在は運用負荷を大幅に下げる

### 6.2 会員管理の視認性 ⚠️ 要検討

会員数が増えたとき（150名〜）のテーブルが重くなる懸念。

**改善提案**:
- ステータス別タブ（全員 / trial / member / inactive）
- サジェスト付き検索
