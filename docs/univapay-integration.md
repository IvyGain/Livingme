# UnivaPay 連携ガイド

> 作成日: 2026-03-23
> 対象: Living Me 管理者・クライアント

---

## このドキュメントについて

Living Me では月額課金の決済手段として **UnivaPay** を使用します。
このドキュメントでは「あなたが準備すること」と「システムがどう動くか」を説明します。

---

## あなたが準備すること（手順）

### ステップ 1：UnivaPay の加盟店アカウントを取得する

UnivaPay を使用するには、まず加盟店として契約が必要です。

1. UnivaPay の担当者（または代理店）に連絡し、加盟店申請を行う
2. 審査完了後、**UnivaPay 管理画面（マーチャントポータル）** へのログイン情報が届く
3. ログインして管理画面に入れることを確認する

> ※ すでにアカウントをお持ちの場合はステップ 2 へ

---

### ステップ 2：アプリトークンを発行する

UnivaPay 管理画面で「アプリトークン」を作成します。これがシステムとの接続キーです。

1. UnivaPay 管理画面にログイン
2. **「設定」→「アプリトークン」**（または API トークン）メニューを開く
3. 「新規作成」をクリック
4. 以下の情報をメモしておく：
   - **アプリトークン**（例: `ap_live_xxxxxxxxxxxxxxxx`）
   - **シークレット**（例: `xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`）

> ⚠️ シークレットは作成時にしか表示されません。必ずメモしてください。

---

### ステップ 3：月額料金を確認する

現在のシステムでは月額 **¥5,500（税込）** が設定されています。
UnivaPay 側でも同じ金額でサブスクリプションが作成されます。
金額を変更する場合は開発者に連絡してください。

---

### ステップ 4：Webhook エンドポイントを登録する

UnivaPay から「支払い成功」「解約」などの通知を受け取るため、Webhook の設定が必要です。

1. UnivaPay 管理画面で **「Webhook」** メニューを開く
2. 新規 Webhook を作成し、以下の URL を登録する：

```
https://（あなたのサイトURL）/api/webhooks/univapay
```

例：本番環境が `https://livingme.example.com` の場合
```
https://livingme.example.com/api/webhooks/univapay
```

3. 通知するイベントとして最低限以下を有効にする：
   - `subscription_payment` （定期支払い成功）
   - `subscription_failure` （支払い失敗）
   - `subscription_canceled` （解約）

> ※ サイトの URL が決まっていない場合は、URLが確定してから設定してください。

---

### ステップ 5：管理画面でキーを登録する

Living Me の管理画面で取得したキーを登録します。

1. `https://（あなたのサイトURL）/admin/settings` を開く
2. **「UnivaPay」** セクションに以下を入力する：

| 項目 | 内容 |
|------|------|
| UnivaPay アプリトークン | ステップ 2 でメモしたトークン |
| UnivaPay アプリシークレット | ステップ 2 でメモしたシークレット |
| UnivaPay 価格（円） | `5500` |

3. 「保存する」をクリック

---

### 設定完了の確認

以下がすべて「登録済み」と表示されれば設定完了です：
- ✅ UnivaPay アプリトークン
- ✅ UnivaPay アプリシークレット
- ✅ UnivaPay 価格

## システムでの動き方

### 会員登録フロー

```
会員が /join ページを開く
    ↓
「登録する」ボタンをクリック
    ↓
UnivaPay の決済フォームが表示される（カード情報入力）
    ↓
支払い成功 → UnivaPay がシステムに通知
    ↓
システムが会員ステータスを「正式会員（アクティブ）」に変更
    ↓
会員がすべてのコンテンツにアクセスできるようになる
```

### 毎月の自動課金フロー

```
毎月の課金日が来る
    ↓
UnivaPay が自動的にカードに請求
    ↓
成功した場合 → 引き続き正式会員のまま
失敗した場合 → UnivaPay がシステムに通知
           → 会員ステータスを「停止中（非アクティブ）」に変更
           → 会員はログイン後に /inactive ページへ案内される
```

### 解約フロー

```
会員が解約（または管理者が停止）
    ↓
UnivaPay がシステムに通知
    ↓
会員ステータスを「停止中」に変更
    ↓
コンテンツへのアクセスができなくなる
```

## データの保存について

支払いに関するデータはすべてデータベースに保存されます：

| データ | 保存場所 | 内容 |
|--------|---------|------|
| サブスクリプション ID | 会員情報（User テーブル） | UnivaPay が発行するサブスク ID |
| 支払いイベント | UnivaPayEvent テーブル | 成功・失敗・解約などの全履歴 |

> アプリトークンやシークレットは **暗号化してデータベースに保存** されます。平文では保存されません。

## よくある質問

**Q. Stripe との違いは？**
A. Stripe は海外製の決済サービスで、UnivaPay は日本国内向けに特化した決済サービスです。日本のカード会社との連携やコンビニ決済などに対応しやすい特徴があります。Living Me では UnivaPay をメインの決済手段として使用します。

**Q. 設定後すぐに使えますか？**
A. ステップ 5 でキーを登録した後、すぐに使用できます。ただし Webhook（ステップ 4）はサイトの本番 URL が決まってから設定してください。

**Q. テスト環境でテストできますか？**
A. UnivaPay はテスト用のアプリトークンを発行できます。本番運用前に開発者と一緒にテストを行うことをお勧めします。
